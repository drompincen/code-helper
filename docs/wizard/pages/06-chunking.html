6 Chunking Strategy

Fixed-size character windows with configurable overlap

How It Works

Algorithm

Window size: 1000 characters (default)

Overlap: 200 characters (default)

Step: size - overlap = 800 characters

Text is sliced into overlapping windows. Each chunk records its character offset (start, end) for precise location.

Why overlap? Ensures that phrases split across chunk boundaries still appear in at least one complete chunk.

Example

// Input: 2500 chars, chunk=1000, overlap=200
Chunk 0: chars 0-1000
Chunk 1: chars 800-1800
Chunk 2: chars 1600-2500
// 3 chunks, 200-char overlap between each

Properties

Stable Order
Same input always produces same chunks
Offset Tracking
char_start and char_end map back to original text
v2 note: Character-based chunking is sufficient. Token-perfect or syntax-aware chunking is a future enhancement.
