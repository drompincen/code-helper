8 Search: Exact, Semantic, Hybrid

Three modes for different search needs — hybrid is the default

Exact Search

EXACT

Case-insensitive substring match across all indexed chunk text.

Score: Occurrence count + density (matches × length / chunk length)

Best for: Annotations (@Transactional), class names, specific identifiers

Semantic Search

SEMANTIC

Embeds the query phrase, computes cosine similarity against all chunk embeddings.

Score: Cosine similarity (0.0 to 1.0)

Best for: Conceptual queries like "how onboarding state is managed"

Hybrid Search

HYBRID

Runs both exact and semantic, normalizes scores, merges by chunk ID.

Score: 0.3 × exact + 0.7 × semantic

Best for: General-purpose search (default CLI mode)
