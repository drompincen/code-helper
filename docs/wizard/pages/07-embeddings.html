7 Embeddings & Vectors

Self-contained TF-IDF hash vectorization — no external API needed

Embedding Pipeline

1
Tokenize
Split on non-alphanumeric, lowercase, filter single chars
2
Remove Stop Words
Filter ~80 common English words (the, is, and, ...)
3
Unigram Features
Hash each token to a vector position. Increment by 1.0
4
Bigram Features
Hash each adjacent token pair. Increment by 0.5
5
L2 Normalize
Scale vector to unit length for cosine similarity

Technical Details

PropertyValue
Dimension256 (configurable)
Hash functionFNV-1a variant
Feature typesUnigrams (1.0) + Bigrams (0.5)
NormalizationL2 (unit vector)
SimilarityCosine (= dot product of unit vectors)
StorageDOUBLE[] column in DuckDB
Why this approach? Self-contained, no API keys, no model downloads. Captures word overlap and bigram context. Good enough for code search in v2.
Limitation: No true synonym understanding. "automobile" won't match "car". For that, a real embedding model (e.g., sentence-transformers) would be needed.
